What You Will Build

A Crime Wave Simulator mod that combines every technique from the previous tutorials into one working mod:

  • An ECS system that triggers crimes on demand
  • A settings panel for crime frequency
  • A hotkey binding to activate/deactivate the crime wave
  • A Harmony patch to modify crime severity
  • Save/load persistence for a crime counter

Prerequisites: Complete the previous tutorials: Harmony Patching, Persisting Custom Data, Custom Hotkeys.

Mod Architecture

The Crime Wave Simulator uses five files, each handling one responsibility:

FilePurposeTechnique
Mod.cs Entry point, Harmony init, system registration IMod, Harmony
CrimeWaveSettings.cs Settings panel + hotkey binding ModSetting, ProxyBinding
CrimeWaveSystem.cs ECS system that triggers crimes GameSystemBase, EntityQuery
CrimeWaveStats.cs Persistent global counter IDefaultSerializable
CrimeSeverityPatch.cs Harmony patch to modify steal amounts HarmonyPrefix

Step 1: Settings with Hotkey

The settings class declares the hotkey (Ctrl+Shift+C) and a configurable crime frequency slider. Players can rebind the key and adjust the frequency in the Options menu. 

// === CrimeWaveSettings.cs ===
using Colossal.IO.AssetDatabase;
using Game.Input;
using Game.Modding;
using Game.Settings;

[FileLocation(nameof(CrimeWaveMod))]
[SettingsUIKeyboardAction("ToggleCrimeWave", ActionType.Button)]
public class CrimeWaveSettings : ModSetting
{
    public CrimeWaveSettings(IMod mod) : base(mod) { }

    /// <summary>
    /// Hotkey to activate/deactivate the crime wave. Default: Ctrl+Shift+C.
    /// </summary>
    [SettingsUIKeyboardBinding(BindingKeyboard.C, "ToggleCrimeWave",
        ctrl: true, shift: true)]
    public ProxyBinding ToggleCrimeWaveBinding { get; set; }

    /// <summary>
    /// How many crimes to trigger per cycle (1-10).
    /// Appears as a slider in the Options panel.
    /// </summary>
    [SettingsUISection("Crime Wave")]
    [SettingsUISlider(min = 1, max = 10, step = 1)]
    public int CrimesPerCycle { get; set; } = 3;

    /// <summary>
    /// Multiplier applied to stolen amounts (0.5x to 5x).
    /// </summary>
    [SettingsUISection("Crime Wave")]
    [SettingsUISlider(min = 0.5f, max = 5.0f, step = 0.5f)]
    public float SeverityMultiplier { get; set; } = 1.0f;

    public override void SetDefaults()
    {
        CrimesPerCycle = 3;
        SeverityMultiplier = 1.0f;
    }
}

Step 2: Mod Entry Point

The entry point initializes Harmony, registers settings and keybindings, and registers all custom systems. Harmony is cleaned up in OnDispose. 

// === Mod.cs ===
using Colossal.IO.AssetDatabase;
using Colossal.Logging;
using Game;
using Game.Modding;
using HarmonyLib;

public class CrimeWaveMod : IMod
{
    internal static ILog Log = LogManager.GetLogger(nameof(CrimeWaveMod))
        .SetShowsErrorsInUI(true);
    internal static CrimeWaveSettings Settings { get; private set; }

    private Harmony _harmony;

    public void OnLoad(UpdateSystem updateSystem)
    {
        Log.Info("CrimeWaveMod loading");

        // 1. Initialize Harmony patches
        _harmony = new Harmony("com.example.crimewavesimulator");
        _harmony.PatchAll(typeof(CrimeWaveMod).Assembly);

        // 2. Initialize settings and hotkey
        Settings = new CrimeWaveSettings(this);
        Settings.RegisterInOptionsUI();
        AssetDatabase.global.LoadSettings(
            nameof(CrimeWaveMod), Settings, new CrimeWaveSettings(this));
        Settings.RegisterKeyBindings();

        // 3. Register ECS systems
        updateSystem.UpdateAt<CrimeWaveSystem>(
            SystemUpdatePhase.GameSimulation);
        updateSystem.UpdateAt<CrimeWaveStats>(
            SystemUpdatePhase.GameSimulation);

        Log.Info("CrimeWaveMod loaded successfully");
    }

    public void OnDispose()
    {
        _harmony?.UnpatchAll("com.example.crimewavesimulator");
        Settings?.UnregisterInOptionsUI();
        Log.Info("CrimeWaveMod disposed");
    }
}

Step 3: ECS System -- Crime Trigger

The core system listens for the hotkey toggle, then periodically triggers crimes on random buildings when active. It uses the game's event command pattern (creating AddCriminal and AddAccidentSite entities) to trigger crimes through the standard pipeline. 

// === CrimeWaveSystem.cs ===
using Game;
using Game.Citizens;
using Game.Common;
using Game.Events;
using Game.Input;
using Game.Prefabs;
using Game.Simulation;
using Unity.Collections;
using Unity.Entities;

/// <summary>
/// Triggers crimes on demand when the crime wave is active.
/// Toggle with the configured hotkey (default: Ctrl+Shift+C).
/// </summary>
public partial class CrimeWaveSystem : GameSystemBase
{
    private ProxyAction _toggleAction;
    private bool _isActive;
    private int _tickCounter;

    private EntityQuery m_CitizenQuery;
    private EntityQuery m_CrimeDataQuery;
    private SimulationSystem m_SimulationSystem;
    private CrimeWaveStats m_Stats;

    protected override void OnCreate()
    {
        base.OnCreate();

        _toggleAction = CrimeWaveMod.Settings.GetAction("ToggleCrimeWave");

        m_SimulationSystem = World.GetOrCreateSystemManaged<SimulationSystem>();
        m_Stats = World.GetOrCreateSystemManaged<CrimeWaveStats>();

        // Query for eligible citizens (unemployed, adult, not already criminal)
        m_CitizenQuery = GetEntityQuery(
            ComponentType.ReadOnly<Citizen>(),
            ComponentType.Exclude<Criminal>(),
            ComponentType.Exclude<Game.Common.Deleted>(),
            ComponentType.Exclude<Game.Tools.Temp>()
        );

        // Query for the crime event prefab
        m_CrimeDataQuery = GetEntityQuery(
            ComponentType.ReadOnly<CrimeData>(),
            ComponentType.ReadOnly<EventData>(),
            ComponentType.Exclude<Locked>()
        );
    }

    public override int GetUpdateInterval(SystemUpdatePhase phase) => 16;

    protected override void OnUpdate()
    {
        // Check hotkey toggle
        if (_toggleAction != null && _toggleAction.WasPressedThisFrame())
        {
            _isActive = !_isActive;
            CrimeWaveMod.Log.Info($"Crime wave {(_isActive ? "ACTIVATED" : "deactivated")}");
        }

        if (!_isActive) return;

        // Trigger crimes every 256 ticks (~4 seconds at 60fps)
        _tickCounter++;
        if (_tickCounter < 16) return;
        _tickCounter = 0;

        TriggerCrimes();
    }

    private void TriggerCrimes()
    {
        using var prefabs = m_CrimeDataQuery.ToEntityArray(Allocator.Temp);
        if (prefabs.Length == 0) return;

        Entity eventPrefab = prefabs[0];
        EventData eventData = EntityManager.GetComponentData<EventData>(eventPrefab);

        using var citizens = m_CitizenQuery.ToEntityArray(Allocator.Temp);
        if (citizens.Length == 0) return;

        int crimesToTrigger = CrimeWaveMod.Settings.CrimesPerCycle;
        int triggered = 0;

        for (int i = 0; i < crimesToTrigger && i < citizens.Length; i++)
        {
            Entity citizen = citizens[i];

            // Create crime event entity
            Entity crimeEvent = EntityManager.CreateEntity(eventData.m_Archetype);
            EntityManager.SetComponentData(crimeEvent, new PrefabRef(eventPrefab));
            EntityManager.GetBuffer<TargetElement>(crimeEvent)
                .Add(new TargetElement(citizen));

            // Create AddCriminal command
            Entity cmd = EntityManager.CreateEntity(
                ComponentType.ReadWrite<Game.Common.Event>(),
                ComponentType.ReadWrite<AddCriminal>()
            );
            EntityManager.SetComponentData(cmd, new AddCriminal
            {
                m_Event = crimeEvent,
                m_Target = citizen,
                m_Flags = CriminalFlags.Robber | CriminalFlags.Planning
            });

            triggered++;
        }

        if (triggered > 0)
        {
            m_Stats.TotalCrimesTriggered += triggered;
            CrimeWaveMod.Log.Info($"Triggered {triggered} crimes " +
                $"(total: {m_Stats.TotalCrimesTriggered})");
        }
    }
}

Step 4: Persistent Crime Counter

The stats system implements IDefaultSerializable to persist the total crime count across save/load. It uses the version byte pattern for safe future upgrades. 

// === CrimeWaveStats.cs ===
using Colossal.Serialization.Entities;
using Game;

/// <summary>
/// Persistent mod-global statistics. Survives save/load.
/// SetDefaults provides clean values for new games.
/// </summary>
public partial class CrimeWaveStats : GameSystemBase, IDefaultSerializable
{
    public int TotalCrimesTriggered { get; set; }
    public int TotalModifiedSeverity { get; set; }

    private const byte kCurrentVersion = 1;

    public void Serialize<TWriter>(TWriter writer) where TWriter : IWriter
    {
        writer.Write(kCurrentVersion);
        writer.Write(TotalCrimesTriggered);
        writer.Write(TotalModifiedSeverity);
    }

    public void Deserialize<TReader>(TReader reader) where TReader : IReader
    {
        reader.Read(out byte version);
        reader.Read(out int crimes);
        TotalCrimesTriggered = crimes;
        reader.Read(out int severity);
        TotalModifiedSeverity = severity;
    }

    public void SetDefaults(Context context)
    {
        TotalCrimesTriggered = 0;
        TotalModifiedSeverity = 0;
    }

    protected override void OnUpdate() { }
}

Step 5: Harmony Patch -- Modify Crime Severity

A Harmony prefix patch intercepts CrimeCheckSystem.OnUpdate to apply the mod's severity multiplier to crime data. This modifies the prefab data so that all crimes created by the game use the adjusted values. 

// === CrimeSeverityPatch.cs ===
using HarmonyLib;
using Game.Prefabs;
using Game.Simulation;
using Unity.Collections;
using Unity.Entities;

/// <summary>
/// Modifies crime steal amounts by applying the settings multiplier
/// to the CrimeData prefab before CrimeCheckSystem processes it.
/// </summary>
[HarmonyPatch(typeof(CrimeCheckSystem), "OnUpdate")]
public static class CrimeSeverityPatch
{
    // Cache original values to restore on unpatch
    private static float _originalAbsoluteMax = -1f;
    private static float _originalAbsoluteMin = -1f;

    [HarmonyPrefix]
    public static void Prefix()
    {
        try
        {
            float multiplier = CrimeWaveMod.Settings.SeverityMultiplier;
            if (Math.Abs(multiplier - 1.0f) < 0.01f) return;

            var em = World.DefaultGameObjectInjectionWorld.EntityManager;
            var query = em.CreateEntityQuery(
                ComponentType.ReadWrite<CrimeData>(),
                ComponentType.ReadOnly<EventData>(),
                ComponentType.Exclude<Locked>()
            );

            using var entities = query.ToEntityArray(Allocator.Temp);
            for (int i = 0; i < entities.Length; i++)
            {
                CrimeData data = em.GetComponentData<CrimeData>(entities[i]);

                // Save original values on first run
                if (_originalAbsoluteMax < 0)
                {
                    _originalAbsoluteMax = data.m_CrimeIncomeAbsolute.max;
                    _originalAbsoluteMin = data.m_CrimeIncomeAbsolute.min;
                }

                // Apply multiplier to steal amounts
                data.m_CrimeIncomeAbsolute.min = _originalAbsoluteMin * multiplier;
                data.m_CrimeIncomeAbsolute.max = _originalAbsoluteMax * multiplier;
                em.SetComponentData(entities[i], data);
            }
        }
        catch (Exception ex)
        {
            CrimeWaveMod.Log.Error($"CrimeSeverityPatch error: {ex}");
        }
    }
}

Defensive Error Handling

Always wrap Harmony patches in try/catch. An unhandled exception in a prefix can prevent the original method from running, potentially breaking game systems. Log the error and let the original continue.

How the Pieces Connect

[Game Start]
    |
    v
Mod.OnLoad()
    |-- Harmony.PatchAll()      --> CrimeSeverityPatch applied
    |-- Settings init           --> Hotkey Ctrl+Shift+C registered
    |-- RegisterKeyBindings()   --> ProxyAction created in InputManager
    |-- Register systems        --> CrimeWaveSystem + CrimeWaveStats
    |
    v
[Player presses Ctrl+Shift+C]
    |
    v
CrimeWaveSystem.OnUpdate()
    |-- WasPressedThisFrame()   --> _isActive = true
    |
    v
[Every 16 ticks while active]
    |
    v
CrimeWaveSystem.TriggerCrimes()
    |-- Create crime event entity (from CrimeData prefab)
    |-- Create AddCriminal command (Robber | Planning)
    |-- Increment CrimeWaveStats.TotalCrimesTriggered
    |
    v
[Game's crime pipeline takes over]
    AddCriminalSystem --> Criminal component on citizen
    CriminalSystem --> Planning -> Preparing -> Crime/Arrest
    AccidentSiteSystem --> Police dispatch
    |
    v
[Meanwhile, on every CrimeCheckSystem.OnUpdate]
    CrimeSeverityPatch.Prefix() --> Multiplies steal amounts
    |
    v
[Player saves game]
    CrimeWaveStats.Serialize() --> TotalCrimesTriggered persisted
    |
    v
[Player loads game]
    CrimeWaveStats.Deserialize() --> Counter restored

Testing Checklist

CS2 mods have no unit test framework -- verification is done in-game with log output:

  1. Build the mod with dotnet build -- auto-deploys to game's Mods folder.
  2. Launch the game and start a city with police stations and residential zones.
  3. Open the game log and filter for "CrimeWaveMod".
  4. Press Ctrl+Shift+C -- verify "Crime wave ACTIVATED" appears in the log.
  5. Wait a few seconds -- verify "Triggered N crimes" messages appear.
  6. Check the Options menu -- verify the CrimesPerCycle slider is visible and functional.
  7. Adjust the SeverityMultiplier slider and verify log shows the patch applying.
  8. Save the game, exit, reload -- verify the crime counter is preserved.
  9. Start a new game -- verify the counter resets to zero (SetDefaults).
  10. Press Ctrl+Shift+C again -- verify "deactivated" and crimes stop.

Extending the Mod

Ideas for further development using the techniques from this tutorial series:

  • Target specific buildings -- Use AddAccidentSite to create crime scenes at specific locations (see Crime Trigger).
  • Custom UI panel -- Add a toolbar button that shows live crime stats (see Mod UI Buttons).
  • Replace CrimeCheckSystem -- For full control over which citizens become criminals, disable the vanilla system and write your own (see System Replacement).
  • Per-building crime counter -- Add a custom IComponentData, ISerializable component to track crimes per building (see Data Persistence).
  • Transpiler patch -- Use a transpiler to change the wellbeing threshold that determines crime probability (see Harmony Transpilers).
