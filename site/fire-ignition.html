Fire Ignition

What: How fires start on buildings and trees in CS2 -- the ignition trigger, hazard probability math, and the system pipeline from spark to blaze.

Why: To build mods that programmatically ignite entities, tweak fire probability, or prevent fires entirely.

Boundaries: Does not cover fire rescue dispatch, fire station pathfinding, or fire visual effects. Fire spread is documented where it overlaps with ignition.

Status: Complete

How It Works

The key insight: building fires and forest fires are not separate systems. They share a single unified pipeline with three systems, each with a distinct job:

  1. FireHazardSystem -- decides whether a spontaneous fire should start (runs every ~68 seconds)
  2. IgniteSystem -- processes ignition requests and attaches the OnFire component to targets (runs every frame)
  3. FireSimulationSystem -- escalates intensity, applies damage, triggers spread (runs every ~1 second)

The only difference between building and tree fires is the hazard calculation path inside FireHazardData.GetFireHazard(). Buildings factor in level, zone type, and fire station coverage. Trees factor in temperature, rainfall, and local effects. After that, the same Ignite event feeds into the same pipeline.

All three systems live in Game.dll. The relevant namespaces:

Data Flow

This is the full path from spontaneous ignition through to destruction. Fire spread creates new Ignite events that feed back into the same loop.

FireHazardSystem (every 4096 frames / ~68s) | | Picks 1-in-64 chunks at random | For each building or tree in chunk: | | Building path Tree path | ---------------- ---------------- | base hazard (prefab) base hazard (prefab) | * level penalty * climate factor | * zone multiplier (temp curve * rain curve) | * fire station coverage * local effect modifier | * district modifier * damage factor ^4 | * damage factor ^4 | | TryStartFire: | probability = hazard * startProbability | roll: random(10000) < probability | v Fire Event Entity created | Has: PrefabRef + Fire tag + TargetElement buffer | Also creates: Ignite entity (Event tag + Ignite data) | v IgniteSystem (every frame) | Deduplicates by target (keeps highest intensity) | Adds OnFire component to target entity | Adds target to event's TargetElement buffer | v OnFire on target entity | intensity starts at 1.0 (default) | v FireSimulationSystem (every 64 frames / ~1s) | +-- Damage: intensity += escalationRate * 1.067 | structural damage += min(0.5, intensity * dt / integrity) | When damage >= 1.0 --> Destroy event | When intensity == 0 --> remove OnFire | +-- Rescue: delay = random(3s-30s) * modifiers | When frame reached --> FireRescueRequest | +-- Spread: roll based on intensity * sqrt(spreadProb) Search nearby entities within spreadRange Success --> new Ignite event (loops back up)

Key Components

Ignite Game.Events

A temporary event component. Created as a standalone entity with the Event tag, processed by IgniteSystem within one frame, then destroyed. This is the primary way to start a fire -- both for spontaneous ignition and for mod-triggered fires.

OnFire Game.Events

Placed on burning entities by IgniteSystem. Persists across save/load. Intensity escalates over time via FireSimulationSystem until the entity is destroyed or the fire is extinguished.

Fire Game.Events

Empty marker component. Tags fire event entities (the event itself, not the burning target). Added to the event entity archetype by Game.Prefabs.Fire.GetArchetypeComponents().

TargetElement Game.Events

Buffer element (IBufferElementData) on fire event entities. Lists all entities currently burning as part of this fire event. Contains a single field: m_Entity.

Forest Fire Climate

Two AnimationCurve fields on the prefab control forest fire likelihood: m_TemperatureForestFireHazard and m_NoRainForestFireHazard. Their product becomes the climate factor. Hot + dry = high factor; cold + wet = near zero. The exact curve shapes are defined in asset data, not in code.

Hazard Calculation

Understanding the math is essential if you want to modify fire probability. Here is what GetFireHazard() does for each path:

Building Hazard

float hazard = prefabFireHazard;                          // DestructibleObjectData.m_FireHazard (or 100)
hazard *= 1f - (buildingLevel - 1) * 0.03f;               // 3% reduction per level above 1
hazard *= zoneFireMultiplier;                              // ZonePropertiesData.m_FireHazardMultiplier
hazard *= math.max(0.01f, 1f - serviceCoverage * 0.01f);  // fire station coverage (never below 1%)
hazard *= (1f + districtModifier);                         // district BuildingFireHazard modifier

float damageFactor = math.max(0f, 1f - totalDamage);
hazard *= damageFactor * damageFactor * damageFactor * damageFactor;  // ^4 dropoff

Tree Hazard

Only applies to wild trees (no Owner component) when naturalDisasters is enabled.

float hazard = prefabFireHazard;                           // DestructibleObjectData.m_FireHazard (or 100)
hazard *= forestFireHazardFactor;                           // temperature curve * no-rain curve
hazard *= (1f + localEffectModifier);                      // ForestFireHazard local effect

float damageFactor = math.max(0f, 1f - totalDamage);
hazard *= damageFactor * damageFactor * damageFactor * damageFactor;  // ^4 dropoff

Final Ignition Roll

After GetFireHazard() returns, TryStartFire() makes the final decision:

float probability = fireHazard * fireData.m_StartProbability;  // hazard * 0.01 by default
bool ignites = random.NextFloat(10000) < probability;          // roll out of 10,000

With default values (hazard=100, startProbability=0.01), probability is 1.0 -- a 0.01% chance per check. Combined with the 1-in-64 chunk skip and 4096-frame interval, actual spontaneous fires are rare.

Examples

Ignite a Building

Create an Ignite event entity. IgniteSystem processes it next frame and adds OnFire to the target. You need a fire event entity first (see helper method below). 

public void IgniteBuilding(Entity targetBuilding)
{
    // Step 1: Create fire event entity from the Building fire prefab
    Entity fireEvent = CreateFireEvent(EventTargetType.Building);
    if (fireEvent == Entity.Null) return;

    // Step 2: Create the Ignite event entity
    Entity igniteEntity = EntityManager.CreateEntity(
        ComponentType.ReadWrite<Event>(),
        ComponentType.ReadWrite<Ignite>()
    );

    // Step 3: Set ignition parameters
    EntityManager.SetComponentData(igniteEntity, new Ignite
    {
        m_Event = fireEvent,
        m_Target = targetBuilding,
        m_Intensity = 1f,          // default start intensity
        m_RequestFrame = 0         // 0 = auto-calculate rescue timing
    });
    // IgniteSystem handles the rest next frame
}

Ignite a Tree

Identical mechanism. The only difference is using EventTargetType.WildTree for the fire event prefab. Programmatic ignition bypasses the naturalDisasters setting check. 

public void IgniteTree(Entity targetTree)
{
    Entity fireEvent = CreateFireEvent(EventTargetType.WildTree);
    if (fireEvent == Entity.Null) return;

    Entity igniteEntity = EntityManager.CreateEntity(
        ComponentType.ReadWrite<Event>(),
        ComponentType.ReadWrite<Ignite>()
    );

    EntityManager.SetComponentData(igniteEntity, new Ignite
    {
        m_Event = fireEvent,
        m_Target = targetTree,
        m_Intensity = 1f,
        m_RequestFrame = 0
    });
    // Trees have 3,000 HP vs 12,000-16,000 for buildings -- they burn fast
}

Create a Fire Event Entity (Helper)

Both examples above need a fire event entity. This helper finds the correct fire prefab by target type and creates the event entity from its archetype. 

private EntityQuery m_FirePrefabQuery;

protected override void OnCreate()
{
    base.OnCreate();
    m_FirePrefabQuery = GetEntityQuery(
        ComponentType.ReadOnly<EventData>(),
        ComponentType.ReadOnly<FireData>()
    );
}

private Entity CreateFireEvent(EventTargetType targetType)
{
    var chunks = m_FirePrefabQuery.ToArchetypeChunkArray(Allocator.Temp);
    var fireDataHandle = GetComponentTypeHandle<FireData>(true);
    var eventDataHandle = GetComponentTypeHandle<EventData>(true);
    var entityHandle = GetEntityTypeHandle();

    foreach (var chunk in chunks)
    {
        var fireDatas = chunk.GetNativeArray(ref fireDataHandle);
        var eventDatas = chunk.GetNativeArray(ref eventDataHandle);
        var entities = chunk.GetNativeArray(entityHandle);

        for (int i = 0; i < fireDatas.Length; i++)
        {
            if (fireDatas[i].m_RandomTargetType == targetType)
            {
                Entity eventEntity = EntityManager.CreateEntity(eventDatas[i].m_Archetype);
                EntityManager.SetComponentData(eventEntity, new PrefabRef(entities[i]));
                chunks.Dispose();
                return eventEntity;
            }
        }
    }

    chunks.Dispose();
    return Entity.Null;
}

Patch Points

ECS Approach (No Harmony -- Preferred)

A custom GameSystemBase can handle most use cases without Harmony:

  • Create Ignite entities to start fires programmatically
  • Query and modify/remove OnFire components
  • Run before IgniteSystem to filter or block Ignite events
  • Modify the FireConfigurationData singleton at runtime

This is safer than Harmony and compatible with Burst-compiled jobs.

Harmony Patches (When ECS Isn't Enough)

TargetPatch TypeWhat It EnablesRisk
FireHazardSystem.OnUpdate() Prefix / Postfix Block all spontaneous fires (Prefix returning false) or modify hazard data before the job runs Low -- only affects spontaneous fire generation
IgniteSystem.OnUpdate() Prefix / Postfix Intercept all ignition attempts (spontaneous, spread, and programmatic). Can selectively filter by entity type. Medium -- central to all ignition; blocking prevents all fires
FireSimulationSystem.OnUpdate() Prefix / Postfix Modify fire behavior post-ignition: escalation rate, spread, rescue timing Medium -- affects all active fires simultaneously
FireHazardData.GetFireHazard() (Building overload) Postfix Per-building hazard modification by type, zone, or district Low -- but this is a struct method; test carefully with Harmony
FireHazardData.GetFireHazard() (Tree overload) Postfix Per-tree hazard modification. Suppress or amplify forest fires by area. Low -- same struct method caveat

Note: The GetFireHazard overloads are called from both FireHazardSystem (spontaneous) and FireSimulationSystem (spread), so patches affect both paths.
