Garbage Collection System

Scope

Question: How does CS2 handle garbage accumulation at buildings, dispatch garbage trucks, collect garbage along routes, and process it at landfills and incinerators?

Verdict: The garbage system is a four-stage ECS pipeline: (1) GarbageAccumulationSystem adds garbage to buildings 16 times per game day based on consumption rates, education, and modifiers; (2) buildings exceeding the request threshold create GarbageCollectionRequest entities; (3) GarbageCollectorDispatchSystem pathfinds and assigns trucks; (4) GarbageTruckAISystem drives routes collecting from all buildings along sidewalks, then unloads at the facility where GarbageFacilityAISystem processes garbage and produces resources.

Out of scope: The garbage infoview UI system, prefab initialization pipeline, and detailed inter-facility transfer dispatch logic.

How It Works

Stage 1: Garbage Accumulation

GarbageAccumulationSystem runs 16 times per game day (update interval = 262144 / (16 * 16) = 1024 frames). Each tick, every building with a GarbageProducer component gains garbage based on its ConsumptionData.m_GarbageAccumulation value.

The accumulation formula considers multiple factors:

// Simplified from GarbageAccumulationSystem.GetGarbageAccumulation()
float baseRate = consumption.m_GarbageAccumulation;

// Education and building level reduce the rate
float adjustedRate;
if (citizenCount > 0)
{
    adjustedRate = max(0, baseRate
        - (buildingLevel * BuildingLevelBalance
        + avgEducation * EducationBalance))
        * citizenCount;
}
else
{
    adjustedRate = baseRate - buildingLevel * BuildingLevelBalance;
}

// Homeless citizens produce extra garbage
adjustedRate += HomelessGarbageProduce * homelessCount;

// District and city modifiers
ApplyModifier(ref adjustedRate, districtModifiers, DistrictModifierType.GarbageProduction);
if (isIndustrial && !isOffice)
    ApplyModifier(ref adjustedRate, cityModifiers, CityModifierType.IndustrialGarbage);

// Per-tick amount (divided by 16 updates/day, rounded randomly)
int garbagePerTick = RoundToIntRandom(adjustedRate / kUpdatesPerDay);

Garbage is capped at GarbageParameterData.m_MaxGarbageAccumulation. When garbage exceeds the warning limit, building efficiency decreases linearly: 

// Efficiency penalty from garbage
float factor = saturate(
    (garbage - WarningGarbageLimit) /
    (MaxGarbageAccumulation - WarningGarbageLimit));
float efficiency = 1.0f - GarbagePenalty * factor;

Stage 2: Collection Request Creation

When a building's garbage exceeds m_RequestGarbageLimit, the accumulation system creates a GarbageCollectionRequest entity: 

// From GarbageAccumulationSystem.RequestCollectionIfNeeded()
if (garbage.m_Garbage > m_GarbageParameters.m_RequestGarbageLimit
    && !validExistingRequest)
{
    garbage.m_CollectionRequest = Entity.Null;
    garbage.m_DispatchIndex = 0;
    Entity e = CommandBuffer.CreateEntity(m_CollectionRequestArchetype);
    CommandBuffer.SetComponent(e, new GarbageCollectionRequest(
        entity, garbage.m_Garbage, flags));
    CommandBuffer.SetComponent(e, new RequestGroup(32u));
}

Industrial zone buildings set the IndustrialWaste flag. When garbage exceeds m_WarningGarbageLimit, a notification icon is added to the building.

Stage 3: Dispatch

GarbageCollectorDispatchSystem runs every 16 frames and follows the standard CS2 service dispatch pattern:

  1. Validate the request: target building still has garbage above the request limit.
  2. Pathfind from SetupTargetType.GarbageCollector (any available truck/facility) to the target building location.
  3. On success, enqueue a VehicleDispatch adding the request to the source facility's ServiceDispatch buffer.
  4. On failure, increment m_DispatchIndex and retry on the next cycle.

Both facilities and trucks can also create reverse requests (ServiceRequestFlags.Reversed) to proactively seek out buildings needing collection. This pathfinds from the truck/facility to a SetupTargetType.GarbageCollectorRequest.

Stage 4: Collection and Processing

GarbageTruckAISystem is the most complex system (~1500 lines). Key behaviors:

  • Route-based collection: Trucks do not just go to the target building. They collect from all buildings along their driving route that have sidewalks. For each road segment, the system finds connected buildings via ConnectedBuilding buffers and collects if garbage exceeds m_CollectionGarbageLimit.
  • Waypoint stops: Path elements marked with CarLaneFlags.Waypoint indicate collection stops. The truck verifies garbage presence before stopping via CheckGarbagePresence().
  • Capacity management: When m_Garbage >= GarbageTruckData.m_GarbageCapacity or the truck is disabled, it returns to the depot. The EstimatedFull flag prevents accepting new dispatches when the estimated load would exceed capacity.
  • Unloading: At the facility, garbage is unloaded at GarbageTruckData.m_UnloadRate per game-second. The garbage is added to the facility's Resource.Garbage economy buffer.
  • Industrial waste: Trucks from industrial-waste-only facilities (GarbageTruckFlags.IndustrialWasteOnly) only collect from industrial zone buildings.

GarbageFacilityAISystem runs every 256 frames and handles the facility side:

  • Processing: Garbage is consumed at ProcessingSpeed / 1024 units per tick, scaled by building efficiency and a fill-level factor: 0.1 + garbageAmount * 1.8 / garbageCapacity.
  • Resource production: Incinerators produce resources (e.g., electricity) via ResourceProductionData buffers proportional to the processing rate.
  • Landfill storage areas: Facilities with SubArea storage distribute garbage between internal and area storage to maintain processing reserves.
  • Inter-facility transfers: Facilities create GarbageTransferRequest entities to send or receive garbage via delivery trucks, enabling garbage flow from landfills to incinerators.
  • Fleet management: Vehicles are enabled/disabled based on available capacity and space. Excess parked vehicles are deleted.

Modding Implications

Adjusting Garbage Rates

  • Modify ConsumptionData.m_GarbageAccumulation on building prefabs to change per-building rates.
  • Use district modifiers (DistrictModifierType.GarbageProduction) for area-wide adjustments.
  • Use city modifiers (CityModifierType.IndustrialGarbage) for industrial-specific adjustments.
  • Harmony-patch GarbageAccumulationSystem.GetGarbageAccumulation() for custom formulas.

Custom Collection Behavior

  • Trucks collect from all buildings along their route, not just the targeted one. This is route-based, not building-based.
  • The collection threshold (m_CollectionGarbageLimit) determines the minimum garbage needed for pickup.
  • Industrial waste is flag-based -- set GarbageCollectionRequestFlags.IndustrialWaste and GarbageTruckFlags.IndustrialWasteOnly.

Facility Customization

  • GarbageFacilityData implements ICombineData -- building upgrades additively stack capacity, vehicle count, and processing speed.
  • m_LongTermStorage = true distinguishes landfills (storage with "full" warning) from incinerators (processing with resource production).
  • Processing speed scales with both building efficiency and current fill level, meaning nearly-empty facilities process slower.

Open Questions

  • Default parameter values: The exact default values for GarbageParameterData fields are set in prefab data (not hardcoded) and would need to be read at runtime or extracted from game prefab assets.
  • Inter-facility transfer dispatch: GarbageTransferDispatchSystem handles matching transfer requests between facilities but was not fully traced.
  • Waste-to-energy integration: GarbagePoweredData suggests some facilities can be powered by garbage, but the exact integration was not fully traced.
  • Service district filtering: Trucks respect service district boundaries via AreaUtils.CheckServiceDistrict(), but the exact district assignment mechanism was not explored.
