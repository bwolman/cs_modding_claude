Crime Trigger

What this covers: How to programmatically trigger crimes in Cities: Skylines II -- who becomes a criminal, how crime events are created, and the full lifecycle from planning through arrest or escape.

Why it matters: If you want to trigger crimes on specific citizens, change crime probabilities, or build custom crime scenarios, you need to understand the event pipeline and state machine that drives it all.

Out of scope: Police vehicle pathfinding, prison management, crime statistics UI.

Status: Complete

How It Works

Crime in CS2 is driven by three systems working together. Understanding the flow between them is the key to modding crime behavior.

The Big Picture

CrimeAccumulationSystem runs in the background, slowly increasing a crime counter on buildings based on zone type and police coverage. This affects police patrol dispatch but does not directly cause crimes.

Actual crimes start in CrimeCheckSystem, which runs once per game day. It scans citizens and picks candidates based on their wellbeing score. When a citizen "rolls" for crime, the system creates a crime event entity and an AddCriminal command entity.

AddCriminalSystem picks up that command on the next frame and attaches a Criminal component to the citizen. From that point, CriminalSystem (running every 16 frames) drives the criminal through a state machine: planning, traveling, committing the crime, and either escaping or getting arrested.

CrimeCheckSystem AddCriminalSystem CriminalSystem (once per game day) (next frame) (every 16 frames) | | | | citizen has low wellbeing | | | + unemployed + not sick | | | + roll succeeds | | | | | +---> Crime Event Entity --------+ | +---> AddCriminal Command -------+--> Criminal component | | added to citizen | | | | Planning ----------+ | Preparing (travel) | | At Target (rob) | | Arrest or Escape |

Who Can Become a Criminal?

CrimeCheckSystem only considers citizens who are:

  • Teen or Adult age (not children or elderly)
  • Unemployed (no Worker component)
  • Not a student
  • Not sick (no HealthProblem component)
  • Not already participating in an active crime

The Wellbeing-to-Crime Curve

Crime probability is driven by Citizen.m_WellBeing (0-100). The conversion uses two different formulas depending on the range:

  • WellBeing 0-25 (low): Linear mapping. t = wellbeing / 25. A wellbeing of 0 gives maximum crime probability.
  • WellBeing 26-100: Quadratic falloff. t = ((100 - wellbeing) / 75)^2. Crime probability drops sharply as wellbeing rises. At 100, it reaches zero.

The resulting t value is interpolated across the occurrence probability range (default 0-50 for first-time, 0-100 for repeat offenders). The final check rolls against a population-scaled random range: max(population / 2000 * 100, 100). Larger cities have lower per-citizen crime rates, but more total crime.

The Criminal State Machine

Once a citizen gets a Criminal component, CriminalSystem drives them through these states via the CriminalFlags bitmask:

Planning -----> Preparing -----> At Target (Robber) (add trip) (traveling) | +----------+----------+ | | Police secured? Not secured | | Arrested Commit crime | (steal money, Jail countdown apply effects) | | Roll prison? TryEscape / \ | Yes No Released | | Sentenced Released | Wait transport | Prisoner | Sentence countdown | Released

The only crime type in the game is Robbery. When a crime succeeds, the criminal steals money from a random renter in the building, and CrimeVictim effects are applied to household members and employees.

Enabled on citizens affected by crime. The m_Effect byte (0-255) reduces wellbeing. Applied to household members (effect = 15) and employees (effect = 5) of the robbed building.

Data Flow

Crime Event Pipeline

[CrimeCheckSystem] ---- once per game day, 16 partitions ---- | | For each eligible citizen: | Roll probability based on wellbeing + population scaling | If roll succeeds: | +---> Create Crime Event Entity | - Crime tag component | - TargetElement buffer (citizen) | - PrefabRef (crime data prefab) | +---> Create AddCriminal Command Entity - m_Event = crime event - m_Target = citizen - m_Flags = Robber | Planning | v [AddCriminalSystem] ---- processes next frame ---- | | Adds Criminal component to citizen entity | Links citizen to crime event's TargetElement buffer | (If citizen already criminal: merges flags) | v [CriminalSystem] ---- every 16 frames ---- | | Planning: Add TripNeeded(Purpose.Crime), transition to Preparing | Preparing: Wait for citizen to arrive at destination | At Target: Create AddAccidentSite(CrimeScene) on building | Crime Scene: Check AccidentSite flags: | Secured? --> Arrested --> Jail --> maybe Prison --> Released | Open? --> Steal money, apply CrimeVictim --> TryEscape --> Released

Crime Accumulation (Background)

[CrimeAccumulationSystem] ---- periodic ---- | | For each building with CrimeProducer: | crime += crimeRate * (10 - policeCoverage) / 10 * coverageFactor | Apply district/city modifiers | | If m_Crime > tolerance (1000): +---> Create PolicePatrolRequest

Examples

Triggering a Crime on a Citizen (Recommended)

This mirrors the game's own approach: create a crime event entity and an AddCriminal command. AddCriminalSystem processes it on the next frame, and CriminalSystem drives the lifecycle from there.

using Game.Citizens;
using Game.Common;
using Game.Events;
using Game.Prefabs;
using Unity.Collections;
using Unity.Entities;

public void TriggerCrimeOnCitizen(EntityManager entityManager, Entity citizenEntity)
{
    // Find the crime event prefab (has CrimeData + EventData).
    var crimeQuery = entityManager.CreateEntityQuery(
        ComponentType.ReadOnly<CrimeData>(),
        ComponentType.ReadOnly<EventData>(),
        ComponentType.Exclude<Locked>()
    );

    using var prefabEntities = crimeQuery.ToEntityArray(Allocator.Temp);
    if (prefabEntities.Length == 0) return;

    Entity eventPrefab = prefabEntities[0];
    EventData eventData = entityManager.GetComponentData<EventData>(eventPrefab);

    // Create the crime event entity from the prefab's archetype.
    Entity crimeEvent = entityManager.CreateEntity(eventData.m_Archetype);
    entityManager.SetComponentData(crimeEvent, new PrefabRef(eventPrefab));
    entityManager.GetBuffer<TargetElement>(crimeEvent)
        .Add(new TargetElement(citizenEntity));

    // Create the AddCriminal command entity.
    Entity cmd = entityManager.CreateEntity(
        ComponentType.ReadWrite<Game.Common.Event>(),
        ComponentType.ReadWrite<AddCriminal>()
    );
    entityManager.SetComponentData(cmd, new AddCriminal
    {
        m_Event = crimeEvent,
        m_Target = citizenEntity,
        m_Flags = CriminalFlags.Robber | CriminalFlags.Planning
    });
}

Creating a Crime Scene at a Building

You can mark any building as an active crime scene to trigger police dispatch. This uses the same AddAccidentSite command pattern that CriminalSystem uses internally.

using Game.Common;
using Game.Events;
using Unity.Entities;

public void CreateCrimeSceneAtBuilding(
    EntityManager entityManager,
    Entity buildingEntity,
    Entity crimeEventEntity)
{
    Entity cmd = entityManager.CreateEntity(
        ComponentType.ReadWrite<Game.Common.Event>(),
        ComponentType.ReadWrite<AddAccidentSite>()
    );
    entityManager.SetComponentData(cmd, new AddAccidentSite
    {
        m_Event = crimeEventEntity,
        m_Target = buildingEntity,
        m_Flags = AccidentSiteFlags.CrimeScene
    });
    // CriminalSystem checks AccidentSite to determine outcome:
    //   Secured     -> criminal gets arrested
    //   Not Secured -> criminal commits robbery and tries to escape
}

Reading a Citizen's Criminal State

using Game.Citizens;
using Unity.Entities;

public string GetCriminalStatus(EntityManager entityManager, Entity citizenEntity)
{
    if (!entityManager.HasComponent<Criminal>(citizenEntity))
        return "Not a criminal";

    Criminal criminal = entityManager.GetComponentData<Criminal>(citizenEntity);
    CriminalFlags flags = criminal.m_Flags;

    if (flags == 0) return "Released (pending removal)";

    if ((flags & CriminalFlags.Prisoner) != 0)
        return $"In prison, {criminal.m_JailTime} ticks remaining";

    if ((flags & CriminalFlags.Arrested) != 0 && (flags & CriminalFlags.Sentenced) != 0)
        return "Sentenced, awaiting prison transport";

    if ((flags & CriminalFlags.Arrested) != 0)
        return $"Arrested, in jail for {criminal.m_JailTime} more ticks";

    if ((flags & CriminalFlags.Planning) != 0)
        return "Planning crime (will start traveling next tick)";

    if ((flags & CriminalFlags.Preparing) != 0)
        return "Traveling to crime target";

    if ((flags & CriminalFlags.Robber) != 0)
        return "At crime target, committing robbery";

    return $"Unknown state: flags={flags}";
}

Calculating Crime Probability from Wellbeing

using Unity.Mathematics;

public float CalculateCrimeProbabilityFactor(byte wellBeing)
{
    float t;
    if (wellBeing <= 25)
    {
        // Low wellbeing: linear. 0 = max crime, 25 = threshold.
        t = (float)wellBeing / 25f;
    }
    else
    {
        // Higher wellbeing: quadratic falloff. Drops sharply.
        t = (float)(100 - wellBeing) / 75f;
        t *= t;
    }
    return t;
}

Patch Points

CrimeCheckSystem.OnUpdate

Patch type: Prefix / Postfix

Enables: Override crime probability calculations, force specific citizens to become criminals, or prevent crime entirely.

Risk: Medium. Changing crime frequency affects police dispatch, statistics, and citizen wellbeing.

CriminalSystem.CriminalJob.Execute

Patch type: Prefix (difficult -- BurstCompiled)

Enables: Modify criminal behavior (skip arrest, change steal amounts, alter state transitions).

Risk: High. BurstCompiled job -- Harmony patches may not work directly. Breaking the state machine can leave criminals in invalid states.

CrimeCheckSystem.CrimeCheckJob.TryAddCrime

Patch type: Prefix (difficult -- BurstCompiled)

Enables: Override the wellbeing-to-crime probability curve.

Risk: High. BurstCompiled. Consider the ECS approach instead -- modify citizen wellbeing directly or create event entities.
